Vulnerabilities

The browser offers different ways to store and exchange data,comes with pros and cons

sensitive data needs to be protected against malicious parties. Javascript runs on browser

and connects to proxy API. Single page aplications are stored an authorization token

in window.localStorage so when you need it, it can be retrieved when sending an

API request. Unfortunately window.localStorage was never designed to be secure.

Which any piece of JavaScript can read and write from/to it, which makes it vulnerable.

Protecting against XSS

In JavaScript applications, an XSS (Cross-Site Scripting) attack is one of the most maliciousattacks.

Browsers offer and implement several features to help prevent those vulnerabilities. A security

header is HTTP Response Header designed for security purposes.

Content Security Policy (CSP)

The CSP header describe a series of directives (or rules) to instruct the browser to allow or block

resources to be loaded and executed. Content-Security-Policy: default-src 'self'; ...directives

Subresource Integrity (SRI)

If an attacker is able to modify the e.g. jquery.min.js file content that is store on a CDN,

our application will still oad the file but it doesn't know the content changed an it might contain

malicious code. In order to prevent that, we can use another important security feature: Subresource Integrity.

Using SRI enables the browser to verify that file. This is how you hash the file 'script src="https://cdnjs.../3.3.1/jquery.min.js" integrity="sha256-FgpCb/KJQ1LfOu91ta32o/NMZxltwRo8QtmkMRdAu8=" crossorigin="anonymous" /script'

Content-Security-Policy is a powerful feature with a lot of options and flexibility to instruct the browser

to understand the context of our application or webisite and take necessary actions. You can control

anything that is going on in the browser.

HTTP Strict Transport Security (HSTS)

This security only allows HTTPS requests on all subsequent request. Enforcing this header highly reduces pissoble

MiTM attacks Strict-Transport-Security: max-age=31536000

Security Sensitive Information

HTTP Cookies, in secure and HttpOnly mode to make your data more secure.

The secure flag ensures that the browser sends the cookie only over a HTTP connection.

The HttpOnly flag prevents any JavaScript code to read from it.
